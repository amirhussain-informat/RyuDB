Snowflake is a cloud multi-tenant service, so a chunk of Snowsight is inapplicable to a local single-user GPU DB: warehouse/compute management, RBAC/users/roles, data sharing & Marketplace, Snowpipe, Cortex
  cloud AI, governance/masking, billing/budgets, replication, Native Apps. I'll mark those N/A and not build them. The win is to match Snowsight on the developer-experience surface (worksheets, results,
  charts, object explorer, data loading, query profile, dashboards, history) and then exceed it with GPU-native features Snowflake structurally can't:

  1. Instant column profiling — click a column → histogram, distinct count, min/max/null% in milliseconds on the GPU (Snowflake charges compute for this).
  2. GPU-kernel-aware query profile — the plan graph shows which operators fired the fused C++/CUDA kernel vs cuDF vs Numba fallback, with timings.
  3. GPU-accelerated inline charts — aggregations for charts computed on-device, so you can chart a million-point result without the browser choking.
  4. Zero-latency local console — no round-trip to a remote warehouse.

  Proposed roadmap

  Phase 0 — Worksheet foundation (DX): multi-worksheet tabs, localStorage persistence + version history, command palette (Cmd+K), dark/light theme, keyboard shortcuts, global object search.

  Phase 1 — Results experience (the heart of "better"): grid sort/filter/resize/reorder/freeze/copy-cell; a Chart tab (bar/line/area/scatter/pie/heatmap) next to Results/Explain; click-to-profile columns
  (GPU-instant histogram/distinct/min/max/null%); multi-result tabs; export +JSON/+Parquet.

  Phase 2 — Object management + data loading: db→schema→table/view tree with DDL/columns/constraints/stats/preview; create/drop/alter UI forms (mirror :alter/:drop); drag-drop data-loading wizard (parquet/csv
  → schema preview → typed CREATE TABLE FROM); schema-aware autocomplete wired to the catalog.

  Phase 3 — Query intelligence + dashboards: visual query-profile graph with per-operator stats + GPU-kernel badges (fused C++ / cuDF / Numba-fallback); overhauled filterable query history with per-query
  details + replay; dashboards (saved queries → chart tiles → filters → auto-refresh); Git-backed worksheet export/import.

  Phase 4 — AI + advanced: NL→SQL assistant (BYO Anthropic/OpenAI key); query diff/compare; SQL+Python notebooks.

  Each phase is a stack of one-PR-per-feature merges (per the autonomy you've granted). Phase 1 is where "better than Snowflake" becomes visible fastest — it builds on the Results component and cursor paging
  that just landed, and the column-profiling + charting are the GPU differentiators.
